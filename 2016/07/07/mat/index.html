最近有项目的服务器老是出现宕机问题，之前没有配置heap dump所以无法定位具体的导致服务器宕机的原因。现在服务器上配置了
heap dump在系统发生崩溃的时候会将系统崩溃时的Java对象和类在heap中的情况记录在一个文件中，文件的格式有多样，目前
我们采用的是.hprof格式的。这个格式的文件需要用MAT（Eclipse Memory Analyzer Tool）内存分析工具来分析出现宕机的原因。

一、理论上分析宕机原因

导致宕机的原因一般有一下两种：
内存溢出（out of memory）：已有的数据超过了其获得到的内存所能存储的范围，比如用一个字节存放1000这个数字就属于内存溢出；
内存泄露（memory leak）：程序由于错误或漏洞造成的内存占用过多，或占用内存后无法释放（很多情况是申请了内存，但程序结束后没有释放）。

内存泄露本质上造成的结果也是内存溢出。只不过一个是内存空间直接导致的，一个是程序的问题间接导致的内存空间不足造成的。

服务器（tomcat）依赖于操作系统和JVM才能在计算机上运行的，tomcat的内存溢出，本质上还是JVM的内存溢出，下面我们简单的了解下Java JVM内存方面的知识。

1、Java内存模型概括


（1）程序计数器（Program Counter Register）
记录线程的执行位置，线程私有内存，唯一一个在Java虚拟机规范中没有规定任何OutOfMemoryError情况的区域。
（2）Java VM 栈（Java Virtual Machine Stacks）
Java虚拟机栈也是线程私有的，声明周期与线程相同。虚拟机栈描述的是Java方法执行的内存模型：每个方法被执行的时候都会同时创建一个栈帧（StackFrame）用于存储局部变量表、操作栈、动态链接、方法出口等信息。每一个方法被调用直至执行完成的过程，
就对应着一个栈帧在虚拟机栈中从入栈到出栈的过程。
异常：
1)StackOverflowError :线程请求的栈深度大于虚拟机所允许的深度时会抛出此异常
2)OutOfMemoryError:虚拟机栈可以动态扩展时，无法申请到足够的内存时会抛出此异常
（3）Native 栈（Native Method Stacks）
为虚拟机使用到的Native方法服务。
异常：StackOverflowError和OutOfMemoryError。
（4）Java 堆（Java Heap）
是Java虚拟机管理的内存中的最大一块儿，是所有程序共享的区域。虚拟机启动时创建，存放对象实例，为对象分配内存。
异常：
OutOfMemoryError：如堆中没有足够的内存给实例分配空间，而且再也无法扩展时会抛出此异常
（5）方法区（Method Area）
是各个线程共享的区域，它用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据。也称为永久代（perm gen）或者非堆（non-heap），就是不属于堆空间的。
这个区域一般不会进行GC。
异常：java.lang.OutOfMemoryError: PermGen space
一般是太多的类信息，比如应用spring很多反射信息，可以适度设置大点。
（6）常量池
运行时常量池（Runtime Constant Pool）是方法区的一部分。Class 文件中除了有类的版本、字段、方法、接口等描述等信息外，
还有一项信息是常量池（Constant Pool Table），用于存放编译期已可知的常量，这部分内容将在类加载后存放到方法区的运行时
常量池中。但是Java语言并不要求常量一定只有编译期预置入Class的常量表的内容才能进入方法区常量池，运行期间也可将新内容放入常量池（最典型的String.intern()方法）。
GC的作用主要是用来回收类和回收常量池，当然有部分方法区，即使永久代（Perm Space）也会一定的回收

2、Java垃圾回收机制

我们知道JVM的内存与GC有着不可分割的关系，现在来了解一下JVM的GC。

JVM是通过generation(分代)来进行垃圾回收的。如下图所示，分为：年轻代（young generation）、年老代（tenured generation）、永久代（permanent generation，简称perm gen）。perm gen（或称Non-Heap 非堆）是个异类，heap空间不包括perm gen。

一般来说，大部分对象都是在年轻代被分配，也在年轻代被回收；当年轻代的内存空间存满时，GC会进行次回收（minor collection），这次回收不涉及到heap中的其他generation，minor collection根据弱年代假设（weak generational hypothesis）来假设年轻代中大量的对象都需要垃圾回收，回收效率很高，速度很快；而年轻代中未被回收的对象会被转移到年老代中，当年老代的内存空间也被对象填满时，会进行主回收（major collector），主回收针对整个heap，回收对象比较多，所以比次
回收慢的多。

1.2 JVM垃圾回收器

JVM有三种垃圾回收器，分别是：

throughput collector：用来做并行young generation回收，由参数-XX:+UseParallelGC启动；
concurrent low pause collector：用来做tenured generation并发回收，由参数-XX:+UseConcMarkSweepGC启动；
incremental low pause collector：可以认为是默认的垃圾回收器。

一般来说，垃圾回收器和回收机制我们自己不会使用，由JVM自己决断。

1.3 Heap中各generation空间的划分

-Xmx：指定最大heap空间
-Xms: 指定最小heap空间
-XX:MinHeapFreeRatio=n：最小剩余空间与活动对象比例
-XX:MaxHeapFreeRatio=n：最大剩余空间与活动对象比例
-XX:NewRation=n：年轻代和年老代空间的比例
-Xmn：Java Heap Young区大小，不熟悉最好保留默认值；
-Xss ：每个线程的Stack大小，不熟悉最好保留默认值；
-XX:PermSize：设定内存的永久保存区域；
-XX:MaxPermSize：设定最大内存的永久保存区域；
-XX:PermSize：设定内存的永久保存区域；
-XX:NewSize：设置JVM堆的年轻代最小空间；
-XX:MaxNewSize：设置JVM堆的‘年轻代最大空间；

年老代的major cllection较慢，所以如果年老代空间小于年轻代空间，则会造成频繁的majorcollection，影响应用的性能。
所以一般来说年老代的空间都比年轻代的空间大。

二、GC Roots

常说的GC(Garbage Collector) roots，特指的是垃圾收集器（Garbage Collector）的对象，GC会收集那些不是GC roots且没有被
GC roots引用的对象。
可以作为GC Roots的对象：
Class - 由系统类加载器(systemclassloader)加载的对象，这些类是不能够被回收的，他们可以以静态字段的方式保存持有
其它对象。
Thread - 活着的线程
Stack Local - Java方法的local变量或参数
JNI Local - JNI方法的local变量或参数
JNI Global - 全局JNI引用
Monitor Used - 用于同步的监控对象
Held by JVM - 用于JVM特殊目的由GC保留的对象，但实际上这个与JVM的实现是有关的。
可能已知的一些类型是：系统类加载器、一些JVM知道的重要的异常类、一些用于处理异常的预分配对象以及一些自定义的类加载器等。然而，JVM并没有为这些对象提供其它的信息，因此需要去确定哪些是属于”JVM持有”的了。

二、shallow size和retained size

在说MAT工具之前，我们先来了解一下shallow size和retained size这两个名词，因为分析的时候会用到。下面对这两个名词做一个详细的解释。

1、shallow size
对象自身占用的内存大小，不包括它引用的对象。
非数组对象：shallow size=对象与它所有的成员变量大小的总和；
数组对象：shallow size=数组元素对象的大小总和。

2、retained size

retained size = 当前对象大小+当前对象可直接或间接引用到的对象大小的总和。
间接引用：A引用B，B引用C，A就是间接引用C。
不过这里值得注意的是retained size的大小需要排除GC Roots直接引用的对象。用图理解会更直观：


如上两图所示：从obj1入手，蓝色代表的是仅仅只有通过obj1才能直接或间接访问的对象，白色代表的是被GC Roots直接引用的对象。
图1中：obj1的retained size = obj1的shallow size + obj2的shallow size+obj3的shallow size；
图2中：obj1的retained size = obj1的shallow size + obj2的shallow size+obj3的shallow size+obj4的shallow size；

二、MAT工具的使用

MAT 不是一个万能工具，它并不能处理所有类型的堆存储文件。但是比较主流的厂家和格式，例如 Sun, HP, SAP 所采用的 HPROF 二进制堆存储文件，以及 IBM 的 PHD 堆存储文件等都能被很好的解析。下面来看看要怎么做呢，也许对你有用。官方文档：
http://help.eclipse.org/luna/index.jsp?topic=/org.eclipse.mat.ui.help/welcome.html

1、MAT工具的安装

和其他插件的安装非常类似，MAT 支持两种安装方式，一种是“单机版“的，也就是说用户不必安装 Eclipse IDE 环境，MAT 作为一个独立的 Eclipse RCP 应用运行；另一种是”集成版“的，也就是说 MAT 也可以作为 Eclipse IDE 的一部分，和现有的开发平台集成。
eclipse安装MAT插件分为以下几步：
