java Annotation

在我们的项目中有用到注解，比如jeecg中的minDao，springMVC等都有用到注解，今天就给大家讲讲注解。

一、概念

1、注解（Annotation）

Java提供了一种原程序中的元素关联任何信息和任何数据的途径和方法。Annotion(注解)是一个接口，程序可以通过反射来获取指定程序元素的Annotion对象，然后通过Annotion对象来获取注解里面的元数据。
Annotation是从jdk5开始引入的。

2、元数据（metadata）

元数据就是数据的数据，主要是描述数据属性（property）的信息，用来支持如指示存储位置、历史数据、资源查找、文件记录等功能。

二、Java中常见的注解

1、JDK系统内置注解

（1）@Override ： 表示重写，子类覆盖父类方法，在写实现接口的类，或继承父类中都能看到

（2）@Deprecated ： 表示这个方法已经过时了，但是也可以引用

（3）@Suppvisewarnings ：用于通知java编译器禁止特定的编译警告，出现警告时用此注解可忽略警告

SuppressWarnings注解的常见参数值的简单说明：
    deprecation：使用了不赞成使用的类或方法时的警告；
    unchecked：执行了未检查的转换时的警告，例如当使用集合时没有用泛型来指定集合保存的类型; 
    fallthrough：当 Switch 程序块直接通往下一种情况而没有 Break 时的警告;
    path：在类路径、源文件路径等中有不存在的路径时的警告; 
    serial：当在可序列化的类上缺少 serialVersionUID 定义时的警告; 
    finally：任何 finally 子句不能正常完成时的警告; 
    all：关于以上所有情况的警告。

2、常见第三方注解

springMVC中的@Autowired、@Service、@Responsebody、@Controller等，具体含义可以学习springMVC，这里就不多做介绍了。

3、注解的分类

（1）按运行机制分类：

源码注解 ：    注解只存在于源码中，编译成.class文件就不存在了
编译时注解：   在源码和class文件中都存在
运行时注解：   在运行阶段还起作用，甚至会影响运行逻辑的注解

（2）按来源分类：

来自JDK注解
第三方注解
自定义注解
元注解 ：注解的注解

（3）根据注解参数的个数分类:

标记注解：一个没有成员定义的Annotation类型被称为标记注解。这种Annotation类型仅使用自身的存在与否来为我们提供信息。比如后面的系统注解@Override;
单值注解
完整注解　

三、自定义注解

在了解了注解之后，我们要学会自定义注解，下面我们来学习自定义注解。

1、元注解

元注解的作用就是负责注解其他注解，Java5.0定义了4个标准的meta-annotation类型，它们被用来提供对其它 annotation类型作说明。

（1）@Target：用于描述注解的作用域，也就是注解可以用在什么地方

取值（ElementType）有:
Constructor      构造方法
Field           字段
Local_variable     局部变量
Method           方法声明
Package           包声明
Parameter          参数声明
Type            类、接口声明

（2）@Retention：用于描述注解的生命周期，也就是说被描述的注解在什么范围内有效

RententionPolicy取值有：
source   在源文件中有效，也就是只在源码中存在，编译时会丢弃
class    在class文件中有效，即编译时会记录到class中，运行时忽略
runtime  在运行时有效，即源码、class文件、运行时都存在，可通过反射读取

（3）@Inherited：标识性注解 ，表示允许子类继承，而实现接口是不能继承的

在网页中画icon小图标

据不完全统计，现在好多网站都使用了icon小图标，比如天猫商城、慕课网商城等。有时候使用适当的图标
能够达到一图胜千言的效果。

虽然天猫商城、慕课网商城都用了icon小图标，但其使用本质不尽相同，下面就介绍三种在网页中画icon小图标的方法：

1、css sprite ：
也称css精灵。一项使用背景图片定位，在网页中画icon小图标的技术

2、font+html
使用font字体用html代码以文本的形式在网页中画icon小图标

3、font+css
实质与第二种方法相同，不同的是，他是将icon小图标对应的html代码结合css属性写成css样式，方便调用

一、css Sprite

在国内被称为“css雪碧”，也叫“css精灵”。它是把网页中一些零星背景图片整合到一张图片文件中，再利用css的背景图片定位到要显示的位置，因此也叫“图片拼合”技术。这样可以减少文件体积和服务器请求次数，从而提高效率。

1、知识点

1、background-image：背景图片属性，引入背景图片；

2、background-position:背景图片定位属性，通过该属性定位小图标该显示的位置

下面是利用这两个属性在网页中画icon小图标的例子：

2、对例子的阐述

.div-red i{
        height: 40px;/*图片的高度*/
        width: 24px;/*图片的宽度*/
        display: block;
        margin-left: 10px;
        margin-right: 8px;
        float: left;
        background-image: url("images/s-icon.png");/*引入背景图片*/
    }

如上所示，样式中定义了要显示的icon小图标的宽度和高度，这个宽度和高度需要和图片中所画的每个图标的宽度和高度相同。也就是说在定义样式时，需要事先规划好背景图片中每个小图标的宽度和高度。

<i style="background-position:0 -40px"></i>

这行代码用了background-positon属性定义了图标显示的位置。对于位置的定义跟图标的宽、高有关。在背景图片中以左上角为（0,0）坐标，依次根据图片宽、高往下或往右写图标的位置。

比如给出的例子中，因为每个图标的宽高都是一样的，所以可以如下这样写：

宽：24px  高：40px
background-position:0 0;/*左上角第一个图片即第一行第一个图片*/
background-position:0 -40px;/*第二行第一个图片*/

background-position:-24px 0;/*第一行第二个图片*/
background-position:-24px -40px;/*第二行第二个图片*/

按照以上所写往后一次类推，可以获取自己想要的图标。需要注意的是，定位中的值只能是负值，如果写成正值的话是不会显示图标的。

综上所述，可以得出一个background-position属性书写的基本公式：

background-position:-|横着前面图标宽度的总和|,-|竖着前面图标高度的总和|；

3、总结

1、css Sprite 画icon小图标的优点：

(1) 相对单个小图标，它节省文件体积和服务器请求次数；

(2) 一般情况下，你需要保存为PNG-24的文件格式（相对PNG-8，PNG-24没有毛边）；

(3) 可以设计出丰富多彩的颜色图标。

2、css Sprite 画icon小图标需要注意：一定要先确认icon小图标的大小，然后再根据小图标的大小来摆放小图标。

3、css Sprite 画icon小图标关键在于：用background-position相对于整张图片定位。

二、font+HTML

使用font字体用html代码以文本的形式在网页中画icon小图标。在这里给大家介绍一个下载图标的网站，有4000以上个免费图标。官方网址：https://icomoon.io

2、知识点

在用font+html在网页中画icon小图标时，我们用到的关键点是@font-face。
@font-face是CSS3中的一个模块，他主要是把自己定义的Web字体嵌入到网页中。

1、@font-face语法

@font-face {
   font-family: <YourWebFontName>;/*设置文本的字体名称*/
   src: <source> [<format>][,<source> [<format>]];/*设置自定义字体的相对路径或者绝对路径，注意，此属性只能在@font-face规则里使用*/
   [font-weight: <weight>];/*设置文本粗细*/
   [font-style: <style>];/*设置文本样式*/
 }

2、案列

如上述代码所示，用@font-face引入字体文件，并写好样式，引入写好的样式，再将图标对应的16进制代码写入html，即可在网页中显示想要的icon小图标了。图标对应的16进制code可在下载文件中的demo.html中找到。

这里值得注意的是，在html中写入图标对应的16进制code时，需要在前面加上“&#x”，否则小图标将不会显示。

3、总结

用font+html在网页中画icon小图标的优点：

1、可根据自己的需要改变图标的颜色或其他css效果
2、可以根据font-size属性来改变图标的大小
3、任意缩放图标不会影响清晰度
4、兼容性较好，支持所有的现代浏览器

3、font+css

font+css是将小图标对应的字体写成css样式，然后引用即可。

log4j的学习及在web项目中的应用

在项目中我们都是使用log4j来控制日志的，下面让我们了解一下其他日志工具并学习一下log4j吧。

一、 日志工具

1、log4j

(一) 简介

Log4j是Apache的一个开放源代码项目，用于记录日志，通过使用Log4j，我们可以控制日志在哪里输出，如控制台、文件、GUI组件、甚至是套接口服务器、NT的事件记录器、UNIX Syslog守护进程等；我们也可以控制每一条日志的输出格式；也可以定义每一条日志信息的级别来更加细致地控制日志的生成过程。这些在使用log4j时是通过一个配置文件来灵活地进行配置，而且不需要修改应用的代码。

此外，通过Log4j其他语言接口，您可以在C、C++、.Net、PL/SQL程序中使用Log4j，其语法和用法与在Java程序中一样，使得多语言分布式系统得到一个统一一致的日志组件模块。而且，通过使用各种第三方扩展，您可以很方便地将Log4j集成到J2EE、JINI甚至是SNMP应用中。

(二)log4j的优点

(1) 可以通过配置文件定义日志级别、输出目的地及输出格式，不用修改应用代码；
(2) 在系统开发阶段可以打印详细的log信息以跟踪系统运行情况,而在系统稳定后可以关闭log输出,从而在能跟踪系统运行情况的同时,又减少了垃圾代码（System.out.println(……)等)；
(3) log4j是一套完备的、灵活的、可配置的日志工具。

2、 Commons Logging

Jakarta Commons Logging (JCL)提供的是一个日志(Log)接口(interface)，同时兼顾轻量级和不依赖于具体的日志实现工具。它提供给中间件/日志工具开发者一个简单的日志操作抽象，允许程序开发人员使用不同的具体日志实现工具。用户被假定已熟悉某种日志实现工具的更高级别的细节。JCL提供的接口，对其它一些日志工具，包括Log4J, Avalon LogKit, and JDK 1.4等，进行了简单的包装，此接口更接近于Log4J和LogKit的实现.

3、 Simple Log

Simple Log是一个logging anti-framework.Simple Log是一个让日记操作变得简单但很小的类库并且几乎不需要你做任何操作就可以得到日记的输出.它与其它日记框架相比最大的特点是使用简单,特别是在条件配置方面.它并不打算在一个包中解决所有日记问题,但它提供足够的功能来满足大多数应用程序所需的日记操作.

4、 MonoLog

Monolog是一个用于监视和日志操作的API,国际化支持.

二、log4j的使用

在使用log4j时，要用到log4j的jar包，我使用的是1.2.x版本。
下载地址：http://logging.apache.org/log4j/1.2/download.html

1、 log4j组件

Log4j 由 logger、appender 和 layout 三个组件组成。可以通过同名的 Java 类访问 Log4j 的这三个组件。

（1） Logger - 日志输出组件

可以通过 log4j API 的 logger 类对其进行访问。它的方法有：debug、info、warn、error、fatal 和 log。这些方法用于记录消息。

（2） Appender - 日志输出目的地组件。

执行日志语句时，Logger 对象将接收来自日志语句的记录请求。此请求是通过 logger 发送至 appender 的。然后，Appender 将输出结果写入到用户选择的目的地。对于不同的日志目的地，提供不同的 appender 类型：

ConsoleAppender - 目的地为控制台的Appender 
FileAppender - 目的地为文件的Appender 
DailyRollingAppender-配置文件每天产生一个log文件
RollingFileAppender - 目的地为大小受限的文件的Appender
WriterAppender - 将日志信息以流格式发送到任意指定的地方
JDBCAppender- 将日志信息保存到数据库中
（3）Layout - 日志格式化组件。

appender 可以用来格式化输出结果的各种布局包括：

简单布局（SimpleLayout）- 包含日志信息的级别和信息字符串
模式布局（PatternLayout）- 可以灵活地指定布局模式
HTML 布局（HTMLLayout）-  以HTML表格形式布局
TTCCLayout  - 包含日志产生的时间、线程、类别等等信息

2、log4j配置

(一) 日志输出级别配置
(1) log4j.rootLogger=日志级别，appender1, appender2, ….

示例：

（1）设置整个工程的日志默认级别，以及日志的输出目标
（2）日志级别为ALL<DEBUG<INFO<WARN<ERROR<FATAL<OFF，不区分大小写。示例中的日志级别为INFO，即只有大于或者等于INFO的日志才会被记录。

(2) log4j.logger.org.mypackage[.MyClass]=日志级别，appender1, appender2, ….

示例：

log4j.rootLogger 中的设置对整个工程生效，若只想对某个包或者某个类生效，可以使用本配置项。

(二) 日志输出目的地配置
1、log4j.appender.appender1.File=文件目录及文件

6、log4j.appender.appender1.layout.ConversionPattern=日志输出格式

例如，%d - %m%n或%d{yyyy-MM-dd HH:mm:ss} %p [%c] %m%n
%c 输出日志信息所属的类的全名
%d 输出日志时间点的日期或时间，默认格式为ISO8601，也可以在其后指定格式，比如：%d{yyy-M-dd HH:mm:ss }，输出类似：2002-10-18- 22：10：28
%f 输出日志信息所属的类的类名
%l 输出日志事件的发生位置，即输出日志信息的语句处于它所在的类的第几行
%m 输出代码中指定的信息，如log(message)中的message
%n 输出一个回车换行符，Windows平台为“rn”，Unix平台为“n”
%p 输出优先级，即DEBUG，INFO，WARN，ERROR，FATAL。如果是调用debug()输出的，则为DEBUG，依此类推
%r 输出自应用启动到输出该日志信息所耗费的毫秒数
%t 输出产生该日志事件的线程名
可参考：http://blog.sina.com.cn/s/blog_4e4dd5570100qowy.html
